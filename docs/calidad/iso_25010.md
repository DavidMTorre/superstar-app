# ISO/IEC 25010 — Calidad del producto software

## Objetivo

Relacionar las características de calidad del estándar **ISO/IEC 25010** con evidencias del proyecto MiniCine Superstar.

## Descripción

ISO/IEC 25010 define el modelo de calidad del producto en **ocho características** principales. A continuación, el mapeo breve:

| Característica | Evidencia en el proyecto |
|----------------|--------------------------|
| **Adecuación funcional** | Suite PHPUnit por módulo (auth, reservas, pagos, tickets, admin). |
| **Rendimiento** | Consultas acotadas con Eager Loading donde aplica; índices en migraciones (FK, únicos). |
| **Compatibilidad** | API versionada (`v1`); cliente desacoplado (React). |
| **Usabilidad** | Layout responsive; tickets legibles en móvil; mensajes de error en español. |
| **Fiabilidad** | Transacciones en pago y validación de ticket; `lockForUpdate` en acceso. |
| **Seguridad** | Sanctum, HMAC en QR, hash de contraseñas, middleware admin. |
| **Mantenibilidad** | Capas separadas; tests automatizados; documentación modular en `/docs`. |
| **Portabilidad** | Laravel y PHP 8.2+; frontend estándar ES modules + Vite. |

## Conclusión

El proyecto puede usarse como **caso de estudio** para auditorías académicas citando ISO 25010 junto con resultados de pruebas en [pruebas/resultados_pruebas.md](../pruebas/resultados_pruebas.md).
